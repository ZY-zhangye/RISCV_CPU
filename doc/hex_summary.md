# hex 回归文件清单

更新时间：2026-07-01

`hex/` 仅保留与当前 Core、已确定后续扩展或 ALU 扩展实验直接相关的物理地址版测试。处理器没有虚拟内存，因此所有 `-v-` 镜像均已删除。

## 保留内容

| 分组 | 数量 | 用途 | 当前状态 |
|---|---:|---|---|
| `rv32ui-p-*` | 42 | RV32I 整数、分支、访存及 FENCE.I | 当前官方回归主体；`ma_data` 暂跳过 |
| `rv32um-p-*` | 8 | RV32M 乘除法 | 当前官方回归主体 |
| `rv32uf-p-*` | 11 | RV32F 单精度浮点 | 为后续 F 扩展保留，当前 Core 尚未实现 |
| `rv32mi-p-csr` | 1 | 基础机器态 CSR/Zicsr | 当前 CSR 回归候选 |
| `rv32mi-p-mcsr` | 1 | 更完整的机器态 CSR 行为 | 为后续 CSR 完善保留，当前不保证全部通过 |
| `rv32uzba-p-*` | 3 | `sh1add/sh2add/sh3add` | 作为现有 ALU 扩展接口的简单 Z 扩展实验候选 |

共保留 66 个 HEX 测试镜像以及筛选后的 `index.csv`。所有镜像基地址为 `0x8000_0000`，索引记录了结束地址、大小和文件名。

## 已删除内容

- 所有虚拟地址版 `-v-` 镜像；
- A、C、D、S-mode、Zfh 测试；
- Zbb、Zbs、Zbc、Zbkb、Zbkx 等更复杂位操作或密码学扩展；
- 与当前回归无关的板级 LED、UART、Timer、旧 `c-test`、聚合 ROM 和转换工具；
- 除 `csr/mcsr` 外的 RV32MI 测试。相关异常行为已经由 Core 级自设测试覆盖。

删除的文件仍可从 Git 历史提交 `3fd59c7` 恢复，不影响追溯。

## 使用方式

统一行为内存通过以下参数加载单个镜像：

```text
+HEX=hex/riscv-tests/rv32ui-p-add.hex
```

官方回归脚本应从 `hex/riscv-tests/index.csv` 读取实际存在的用例，不再扫描或引用已删除分组。
