## 3. 关键设计决策

- Issue Arbiter 从单拍贪心路径重构为 P0 proposal、P1 selected mask、P2 revalidate 的
  三级结构；最终 grant 前使用 ROB ID 防止 stale proposal 误授权新候选。
- Issue Queue 的 visible candidate 在未获 grant 时保持稳定，为多拍全局仲裁提供合法
  valid/ready 语义。
- `issue_uop_t` 与 `execute_uop_t` 补入 `pc`、`pred_taken`、`pred_target`、
  `checkpoint_id`，Dispatch 已完成透传。
- PRF 使用偶/奇双 Bank，每 Bank 三份单读副本；每 Bank 仅接受一个最终 WB，ready bit
  只由实际写入口置位，allocation clear 同 PRD 时优先。
- Operand Read 对齐同步 PRF 返回并提供双 WB bypass；四个执行端具有独立 metadata 与
  response holding，阻塞端首次停顿时本地保存 PRF 数据，不阻塞其他端口。
