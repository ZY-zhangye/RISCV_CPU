## 3. 关键设计决策

- Issue Arbiter 从单拍贪心路径重构为 P0 proposal、P1 selected mask、P2 revalidate 的
  三级结构；最终 grant 前使用 ROB ID 防止 stale proposal 误授权新候选。
- Issue Queue 的 visible candidate 在未获 grant 时保持稳定，为多拍全局仲裁提供合法
  valid/ready 语义。
- `issue_uop_t` 补入 `pc`、`pred_taken`、`pred_target`、`checkpoint_id`。
- `execute_uop_t` 补入 `pc`、`pred_taken`、`pred_target`、`checkpoint_id`、`need_rs1`、
  `need_rs2`；Dispatch 与 Operand Read 已完成透传。
- PRF 使用偶/奇双 Bank，每 Bank 三份单读副本；每 Bank 仅接受一个最终 WB，ready bit
  只由实际写入口置位，allocation clear 同 PRD 时优先。
- Operand Read 对齐同步 PRF 返回并提供双 WB bypass；四个执行端具有独立 metadata 与
  response holding，阻塞端首次停顿时本地保存 PRF 数据，不阻塞其他端口。
- INT0 为单周期 ALU + 1-entry completion buffer，完整移位只放在 INT0。
- INT1/Branch 支持简单 ALU、条件分支、JAL/JALR；`branch_resolve_t` 为一拍寄存脉冲，
  recovery_controller 后续统一消费。
- Writeback Arbiter 使用每 producer 2-entry skid buffer、固定优先级双发选择和寄存输出。
  先不引入 round-robin 反馈环；如果系统级测试暴露饥饿，再引入小型寄存 age 机制。
