# 2026-07-06 Rename/ROB 与 Commit/CSR 集成交接

## 1. 当前版本

- 仓库：`F:\RISCV_CPU`

- 基线提交：`388daef Integrate rename allocation checkpoints`

## 2. 本版本完成内容

- ROB 新增精确异常 `exception_flush/done`，可抢占 branch clear/restore scan。
- RAT/AMT 导出只读 AMT map，供 Free List 异常重建。
- `rename_rob_cluster` 闭合 Decode→Rename→资源分配→ROB 原子 fire、ROB entry 构造、
  branch tail restore 和四路 sticky recovery ack。
- 去除 ROB ready 反向依赖 alloc valid 的组合环。
- `rob_alloc_t` 增加 CSR/system command、prepared operand 与原始指令字段。
- `commit_csr_cluster` 实现合法 CSR、非法 CSR、MRET、ECALL/EBREAK、FENCE 的 ROB-head
  提交语义，并保留普通双提交与 Store 两阶段协议。
- CSR 固定由 Issue Arbiter 路由到 INT0；INT0 只捕获 rs1，ROB completion 保存并前递
  `csr_operand`，执行阶段不修改 CSR、不写 PRF。
- `commit_csr_prf_cluster` 闭合 CSR 提交写与 PRF ready 握手；普通 WB 活跃时 CSR 原子
  提交暂停，避免同 Bank 写回丢失。
- `commit_recovery_cluster` 集成 Rename/ROB、Commit/CSR/PRF 与 Recovery Controller，
  闭合 commit map/reclaim、PRF ready clear、四路恢复 ack 和最终 redirect。
- Commit reclaim 增加 valid/ready 反压，CSR 状态修改增加独立 commit 使能；恢复 busy
  期间抑制重复异常副作用。
- 新增 `backend_int_cluster` 作为首个后端整数集成边界，串接 Commit/Recovery、
  Dispatch Buffer、INT Issue Queue、Issue Arbiter、Operand Read、INT0、INT1/Branch
  和 Writeback Arbiter。当前阶段 LSU/MDU 分派保持反压，仅开放 INT/Branch/CSR loop。
- `backend_int_cluster` 在分支解析和 ROB completion 之间加入一项 resolve 缓冲：分支
  recovery 只在匹配 ROB completion 已经可见时进入 recovery controller，避免误预测恢复
  扫描抢占并丢弃分支自身完成位。
- INT0 CSR prepare 修正立即数型 CSR operand：`CSR_RWI/RSI/RCI` completion data 使用
  5-bit zimm 零扩展，寄存器型 CSR 继续使用 rs1；CSR prepare 不提前写 PRF。
- 新增 `backend_mdu_cluster`，在冻结的 Backend LSU 边界上打开 MDU Issue Queue、
  Operand Read MDU 端口、`muldiv_frontend` 和 Writeback Arbiter 的 MUL/DIV producer。
  `issue_arbiter.mdu_accept_i` 在该集群固定为 `1'b1`，MUL/DIV 实际反压由
  Operand Read 的 MDU holding register 通过 `mdu_ex_ready` 吸收，避免子单元 ready
  组合回全局发射仲裁器。
