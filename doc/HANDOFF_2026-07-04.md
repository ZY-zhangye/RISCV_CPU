## 3. 关键设计决策

- Free List 已从旋转索引/长 carry 链改为更适合 FPGA 的分组选择结构，当前时序安全。
- Branch Predictor update 路径已拆为 U0/U1 两拍，并加入同 index update bypass；暂不继续做 BHT row 化。
- ROB 采用 32 entry、16 row × 2 bank 的逻辑组织，ROB ID 为 `{row[3:0], bank}`。
- Dispatch Buffer 为 6 项固定槽环形队列，lane1 不越过阻塞的 lane0，recovery 整 buffer flush。
- Issue Queue 为固定槽字段数组，双 push、双 wakeup；select 已拆成 S0 pair winner + S1 group candidate，牺牲约 1 拍 candidate 可见延迟换时序余量。

## 4. 下一步任务

下一项按目录应实现：

1. `rtl/issue/issue_arbiter.sv`
2. `test/tb_issue_arbiter.sv`

设计要求见 `doc/rtl_module_specs/09_dispatch_issue.md` 第 5 节和 5.1 节。实现原则：

- 只接收各 IQ 已寄存 candidate，不直连 IQ 内部长组合逻辑。
- 全局最多 3 发射。
- INT 最多 2 条，Branch 只能进 INT1。
- LSU 最多 1 条，MDU 最多 1 条。
- PRF 每个 bank 每周期最多 3 个读源，只统计 `need_rs1/need_rs2`。
- 仲裁输出必须寄存后再驱动 PRF Read / operand_read。
- 如果 5 ns OOC WNS 小于 +0.3 ns，优先继续拆流水，而不是保留单周期复杂仲裁。

完成 `issue_arbiter` 并通过 directed test 后，停下向用户汇报，等待用户后台综合结果。
